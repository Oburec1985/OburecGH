# Изменения plgEvalFRF за 18.05.2026

## Форма списка резонансов

- Чекбокс `Экстремумы` на главной форме открывает форму `uPeakFrm`.
- В `uPeakFrm` выводится таблица резонансных пиков: частота, амплитуда, декремент, индекс отсчета.
- Расчет резонансов перенесен в `uPeakFrm`, чтобы не раздувать код главной формы.
- Таблица перестраивает число строк под найденные пики.
- Резонансы в преднастроенной полосе подсвечиваются зеленым.
- Резонансы около границы полосы подсвечиваются бледно-желтым.
- Для формы добавлена индикация полноты найденных полос: зеленая, если все полосы найдены, красная, если хотя бы одна полоса не найдена.
- Сообщение о незаданном триггере заменено на цветную метку на форме, чтобы не плодить одинаковые `ShowMessage` в runtime.

## Отладка резонансов

- Добавлена временная кнопка логирования расчета пиков.
- Временные строки лога помечены комментариями `CODEx DEBUG`, чтобы их было проще удалить после окончания отладки.
- В лог пишется номер удара и промежуточные данные поиска пиков.

## Серийный номер лопатки

- При сохранении отчета проверяется, назначен ли серийный номер выбранной лопатке.
- Если лопатка выбрана, а серийный номер пустой, поле `SnEdit` подсвечивается розовым.
- Отчет не создается, пока серийный номер не назначен.
- Добавлен обратный сценарий: можно ввести известный серийный номер, нажать Enter, и программа найдет соответствующую лопатку.
- При смене номера лопатки через spin-кнопку серийный номер записывается только если он не пустой и еще не занят другой лопаткой.

## Отчет Excel

- На главную форму добавлено поле `FPressFE` для силы зажатия.
- В Excel-отчет добавлена строка под серийным номером: слева метка `F`, справа значение силы зажатия.

## Режим без датчика силы

- В настройках добавлен чекбокс `NoTahoCb` (`Без датчика силы`).
- В этом режиме отдельный тахо/силовой датчик не используется.
- В качестве тахо берется первый датчик обратной связи из списка сигналов.
- Для режима без датчика силы принудительно используется расчет спектра `SPM`; FRF и Coh не считаются.
- При включенной галке родительский узел показывает `Без датчика силы`.
- При снятии галки выбранный в `TahoNameCB` датчик записывается обратно в тахо-узел, включая имя датчика.
- При выборе датчика в `TahoNameCB` и снятой галке родительский узел показывает имя выбранного тахо.

## Поиск удара в режиме без датчика силы

- Поиск удара работает по тому же автомату состояний, что и обычный режим: `TrOff`, `TrRise`, `TrFall`, `TrEnd`.
- Для режима без датчика силы порог применяется к отклонению сигнала от базового уровня блока.
- Базовый уровень считается как среднее по началу текущего буфера, чтобы убрать постоянное смещение канала обратной связи.
- Триггерный уровень остается пользовательской настройкой формы, без захардкоженных значений в коде.
- Для режима без датчика силы сохраняется отдельный ключ `NoTahoThreshold`; если его еще нет, используется старый пользовательский `Threshold`.
- Добавлен временный лог `NO_TAHO_DEBUG` с состоянием автомата, уровнем базы, максимальным отклонением и фактом завершения удара.

## Временные отладочные места

В коде остались временные строки для диагностики:

- `DebugNoTahoLog` в `forms/uEvalFRFFrm.pas`.
- строки `NO_TAHO_DEBUG` в поиске удара.
- строки `PEAK_DEBUG` в расчете резонансов.

После завершения проверки их можно удалить отдельной уборочной правкой.

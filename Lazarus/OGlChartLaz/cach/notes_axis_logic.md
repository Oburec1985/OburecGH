# Заметки по логике осей OGlChart

## Проблема 1: Рефакторинг осей
Изначально в лазарус-версии чарта ось имела свойство `Orientation` (`caoX` или `caoY`), и для настройки X-координат на страницу добавлялся отдельный объект оси X. 
Тренд ссылался на эту ось через свойство `XAxis`.
Это усложняло структуру дерева и приводило к дублированию.

## Решение 1
1. Убираем `Orientation` из оси. Теперь ось всегда является вертикальной (Y).
2. Настройки X-координат по умолчанию задаются на странице (`TChartPage`): `XMinValue`, `XMaxValue`, `XScale`.
3. У оси Y добавляется признак `UseOwnX: Boolean` и собственные настройки `XMinValue`, `XMaxValue`, `XScale`.
4. Серия (тренд) рендерится по X с использованием настроек X своей родительской оси (если `UseOwnX` включен) или настроек X страницы (если выключен).
5. Удаляем `XAxis` из `cBaseTrend`.

## Проблема 2: Пробел в середине/конце максимумов при первой отрисовке
В русской локали Windows разделителем дробной части является запятая `,`. В рендерере (`uOglChartRenderer.pas`):
1. Метод `FormatTick` при `lDigits = 0` формировал строку формата `'0.'`. Из-за этого число `10` форматировалось как `'10,'` (с запятой на конце).
2. Символ `,` отсутствовал в словаре глифов `GlyphRow`, поэтому отображался как пробел (пустое место). Число `1` форматировалось как `1,0` (при шаге `0.2`) и отображалось как `1 0` (с пробелом посередине).

## Решение 2
1. Исправить `FormatTick`: если `lDigits = 0`, использовать формат `'0'` (без точки на конце).
2. Добавить поддержку символа `,` (запятая) в метод `GlyphRow` рендерера для корректного отображения.

## Результаты доработок
Все задачи по рефакторингу и исправлению ошибок отрисовки успешно завершены:
1. `uOglChartAxis.pas` изменен: удалено перечисление `TChartAxisOrientation` и свойство `Orientation`, добавлены поля `UseOwnX`, `XMinValue`, `XMaxValue`, `XScale`.
2. `uOglChartPage.pas` изменен: добавлены настройки шкалы X по умолчанию для страницы (`XMinValue`, `XMaxValue`, `XScale`).
3. `uOglChartTrend.pas` изменен: удалено устаревшее свойство `XAxis` из `cBaseTrend` и его инициализация.
4. `uOglChartRenderer.pas` изменен:
   - В `GlyphRow` добавлен рендеринг запятой `,`.
   - В `FormatTick` исправлен формат целых чисел (при нуле знаков после запятой).
   - Внедрены новые методы `GetPrimaryXAxis`, `BuildXTicks`, `XValueToPixel` для гибкого масштабирования.
   - Метод `DrawGrid` теперь строит сетку на основе X-настроек страницы (или первой оси Y, если задано `UseOwnX`).
   - Метод `DrawAxes` рисует единую горизонтальную ось внизу и все зарегистрированные вертикальные оси Y.
   - Все текстовые метки (как Y-осей, так и новой шкалы X страницы/оси) поддерживают прямое визуальное редактирование с клавиатуры.
5. Тестовое приложение `Test_component/unit1.pas` успешно адаптировано: удалено создание фиктивной оси X, настройки шкалы X вынесены на уровень страницы, вызовы `AddLine` упрощены.
6. Кодировка всех измененных Delphi-файлов приведена к `Windows-1251`, переводы строк сохранены в формате `CRLF`, файлы не содержат `BOM`.

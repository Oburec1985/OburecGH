# Заметки по оптимизации и исправлению рендеринга OglChart (Lazarus)

## Проблема с исчезновением текста во флажках (TChartFlagLabel)
При включении текстурного атласа (шейдерного рендеринга) текст во флажках полностью переставал рисоваться. На экране отображались пустые белые прямоугольники.

### Причина
В методе `DrawTextLabel` модуля `uOglChartRenderer.pas` перед выводом каждой строки текста стояла проверка:
```pascal
if lLineY + lFont.TextPixelHeight > lLabelRect.Bottom - 2 then
  Break;
```
По умолчанию высота флажка `Height = 30` пикселей.
Высота шрифта `cfAxisLabel` при масштабе 1.55 составляет `TextPixelHeight = 25` пикселей.
Смещение первой строки `lLineY = lLabelRect.Top + 4`.
Таким образом, `lLineY + TextPixelHeight = lLabelRect.Top + 29`, в то время как `lLabelRect.Bottom - 2 = lLabelRect.Top + 28`.
Условие `29 > 28` выполнялось на первой же итерации цикла, приводя к мгновенному прерыванию отрисовки (`Break`).

При отключенном текстурном атласе `TextPixelHeight` рассчитывался программно как `Round(12 * 1.55) = 19`, поэтому текст влезал и рисовался.

### Решение
Изменено условие прерывания отрисовки в `uOglChartRenderer.pas`. Теперь первая строка (`I = 0`) гарантированно выводится на экран:
```pascal
if (I > 0) and (lLineY + lFont.TextPixelHeight > lLabelRect.Bottom - 2) then
  Break;
```
После компиляции и перезапуска тестового проекта `project1_test.exe` текст на флажках (например, "Trend: 0,740") отображается корректно и аккуратно.

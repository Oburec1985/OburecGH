# Заметки о стабилизации рендеринга OGlChart

## 1. Проблема двойной зацикленной линии (Double-Line Artifact)
- **Симптом**: При отображении 1D-буфера (`cBuffTrend1d`) на `PageBars` при приближении/сжатии или при смене масштаба возникал визуальный дефект в виде зацикленной ("двойной") красной линии.
- **Причина**: В вершинном шейдере `SHADER_LINE_LG_1D_VERT` координата X вычислялась на основе `gl_VertexID`. В Immediate Mode OpenGL при частых вызовах отрисовки списков `gl_VertexID` может работать некорректно или сбрасываться, приводя к неверной X-координате на вершинах с четными/нечетными индексами.
- **Решение**: Перевели вычисление X-координаты на использование входного атрибута вершины `gl_Vertex[1]`, который передает правильное мировое или нормализованное значение. Это полностью устранило дефект.

## 2. Рефакторинг инициализации графиков
- Вся логика настройки страниц (`cBasePage`), осей (`cAxis`), серий данных (`cTrend`), буферов (`cBuffTrend1d`, `cBuffTrend2d`) и флагов (`TChartFlagLabel`) была вынесена из ядра графического компонента во внешнюю процедуру инициализации.
- Местом инициализации определена процедура `TForm1.FormCreate` тестового проекта (`unit1.pas`).
- Настройка разбита на логические блоки с подробными комментариями на русском языке для создания наглядного обучающего шаблона использования библиотеки.

## 3. Разделение графиков производительности (100k точек)
- Ранее графики 1D и 2D буферов на странице производительности генерировали абсолютно идентичные значения синуса, из-за чего зеленый график перекрывал красный.
- Смещено смещение по оси Y:
  - Для 1D буфера: Y = Sin(X/1000) * 0.3 + 0.4 (диапазон 0.1 - 0.7)
  - Для 2D буфера: Y = Sin(X/1000) * 0.3 + 0.6 (диапазон 0.3 - 0.9)
- Это позволило визуально контролировать корректность отрисовки обеих кривых на одном экране.
- Корректность подтверждена автоматическим снимком экрана `perf_isolated.png`.

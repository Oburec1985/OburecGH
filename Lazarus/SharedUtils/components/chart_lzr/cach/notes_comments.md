# Технические заметки по документированию OGlChartLaz

## Кодировка файлов и EOL

Все Delphi-файлы (`.pas`, `.dfm`) в этом компоненте должны строго сохраняться в кодировке **Windows-1251 (CP1251)** с окончаниями строк **CRLF**.
Для безопасного редактирования с сохранением русских комментариев используется следующий паттерн через PowerShell:
1. Создание промежуточного UTF-8 файла (`filename.pas.utf8`).
2. Чтение его как UTF-8 и перезапись в Windows-1251 с приведением всех переводов строк к CRLF:
   ```powershell
   [System.IO.File]::WriteAllText('filename.pas', [System.IO.File]::ReadAllText('filename.pas.utf8', [System.Text.Encoding]::UTF8).Replace([string]([char]13 + [char]10), [string][char]10).Replace([string][char]10, [string]([char]13 + [char]10)), [System.Text.Encoding]::GetEncoding(1251))
   ```

## Ключевые архитектурные решения

1. **TOglChart** — центральный компонент, наследующий `TGLWidget` (или аналогичный LCL OpenGL-компонент). Является агрегатором для:
   - **TChartModel** — дерево объектов (страницы, оси, тренды, серии линий).
   - **TOpenGLChartRenderer** — отвечает за непосредственную отрисовку.
   - **TChartFrameListener** — диспетчер событий мыши и клавиатуры.

2. **Интерактивное редактирование меток**:
   - В `TOpenGLChartRenderer` реализовано редактирование числовых значений осей и границ страниц "на месте" (in-place).
   - При отрисовке меток осей через `DrawEditableText` регистрируются прямоугольники попадания `TChartTextHit` в `fTextHits`.
   - В `MouseDown` проверяется попадание клика в зарегистрированные метки, активируется режим редактирования (`fActiveHit.Kind <> celNone`), накапливается текст ввода клавиатуры в `fEditText`.
   - При вводе данных значение конвертируется через `TryStrToFloat` и обновляет свойства соответствующей оси или страницы (`MinValue`, `MaxValue` и т.д.).

3. **Алгоритмы шкал (линейная и логарифмическая)**:
   - Шаг сетки вычисляется автоматически методом `NiceStep`.
   - Построение тиков (делений) разделено на `BuildLinearTicks` и `BuildLogTicks`.
   - Логарифмические шкалы `casLog10` оперируют значениями в степени 10, предотвращая передачу неположительных чисел (используется ограничение `Max(Value, 1e-10)`).

4. **Шейдерный конвейер**:
   - Используются вершинные шейдеры `SHADER_LINE_LG_VERT` и `SHADER_LINE_LG_1D_VERT` для ускорения рендеринга больших массивов точек.
   - Шейдеры осуществляют логарифмическое масштабирование непосредственно на GPU по формулам:
     $$rate = \frac{\log_{10}(x) - \log_{10}(Min)}{\log_{10}(Max) - \log_{10}(Min)}$$
     после чего переводят координаты в пространство отсечения OpenGL.

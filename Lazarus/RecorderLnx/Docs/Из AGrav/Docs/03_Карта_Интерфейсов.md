# RecorderLnx: карта интерфейсов Windows Recorder

Источник: `D:\works\OburecGH\recorder\SharedRUnits\interfaces`.

Эта карта нужна, чтобы при проектировании RecorderLnx не дублировать уже найденные сущности и не терять функционал Windows Recorder. Это не требование переносить COM API один к одному. Это функциональный каталог.

### Правило перехода Record -> Preview

- `TRecorderStateMachine.StartPreview(rscManual)` разрешен из состояния `rsRecord`.
- Смысл перехода: запись текущего кадра/каталога завершается, но режим сбора и просмотра остается активным.
- Следующий `StartRecord` должен создаваться уже как новый кадр записи (`0001`, `0002`, ...); сама файловая логика будет реализована отдельным storage/session layer поверх state machine.

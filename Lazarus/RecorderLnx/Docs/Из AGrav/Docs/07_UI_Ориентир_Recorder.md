# RecorderLnx: UI-ориентир по скриншоту Recorder

Источник: скриншот Recorder из обсуждения 2026-05-25. Сам скриншот приложен в диалоге; при переносе в Obsidian как файл его нужно положить рядом с этой заметкой или в папку ресурсов проекта.

## Что берем как ориентир

Основная форма Recorder устроена как рабочее окно оператора, а не как страница настроек:

- сверху панель вкладок/активных формуляров отображения;
- центральная область занята активным формуляром: таблица, график, мнемосхема или другой визуальный экран;
- справа узкий пульт управления: состояние, время режима, кнопки Stop/Preview/Record, поиск тега, список тегов;
- кнопка входа в настройки открывает подробный диалог настройки функций, включая создание/редактирование формуляров;
- сохранение текущего конфига вынесено отдельной кнопкой;
- дополнительные окна могут создаваться плагинами и жить поверх/рядом с главным окном.

## Что не делаем сейчас

- Не строим полноценный редактор формуляров.
- Не строим дерево устройств и список реальных каналов.
- Не подключаем плагины.
- Не делаем итоговую графику/OpenGL.

Текущий UI должен только закрепить каркас: верхние вкладки формуляров, центральная область, правый пульт состояния и команд, кнопка настроек, сохранение конфига и список тегов-заглушек.

## Поведение Preview и Record

Уточнение по режимам:

- `Preview` запускается только вручную пользователем или внешней программой/плагином. В состоянии Stop данные не идут, поэтому нельзя проверить уровень/trigger без потока данных.
- `Record` использует настройки старта из `TRecorderRunControlSettings`. Если старт ручной, запись начинается сразу. Если старт по уровню/TTL/времени, переход идет в `RecordArmed`, а фактический старт — через событие условия.

## Конфигурации

Нужно различать два уровня конфигурации.

### Глобальная настройка программы

Глобальная настройка относится к экземпляру приложения RecorderLnx, а не к конкретному проекту. Примеры:

- список плагинов;
- путь к дефолтному проектному конфигу;
- шрифты;
- тип времени: ПК или UTC;
- параметры запуска из командной строки.

Глобальная настройка загружается по умолчанию или выбирается через параметры строки запуска. Разные ярлыки могут стартовать RecorderLnx с разными проектными конфигурациями.

### Конфигурация проекта

Конфигурация проекта — это всегда каталог, а не один файл. Внутри каталога лежат отдельные файлы:

- настройки GUI;
- настройки плагинов;
- аппаратная конфигурация;
- настройки запуска/остановки;
- будущие формуляры/мнемосхемы;
- прочие доменные файлы проекта.

Первая структура для разработки:

- `D:\works\OburecGH\Lazarus\RecorderLnx\config\app.ini` — глобальная настройка приложения;
- `D:\works\OburecGH\Lazarus\RecorderLnx\config\projects\default\` — дефолтная конфигурация проекта;
- `D:\works\OburecGH\Lazarus\RecorderLnx\config\projects\default\run-control.ini` — настройки запуска/остановки.

Эта структура предварительная и нужна для первых тестов. Позже путь проектного конфига должен задаваться из `app.ini` или из командной строки.

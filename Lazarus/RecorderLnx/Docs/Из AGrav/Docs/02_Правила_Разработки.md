# Правила Разработки RecorderLnx

В целях исключения дублирования информации, все специфичные для проекта RecorderLnx технические и организационные правила разработки консолидированы в локальном файле документации репозитория `OburecGH`:

👉 **[development-rules.md (в репозитории OburecGH)](file:///c:/Oburec/OburecGH/Lazarus/RecorderLnx/Docs/development-rules.md)**

---

### Основные разделы локальных правил разработки:
- **Кодировки файлов**: UTF-8 для переведённых UI-модулей (`{$codepage UTF8}`), восстановление через `cach/_fix_umainform_encoding.py`. Подробно: `Docs/source-encoding.md`, AGrav `10_Кодировка_Исходников.md`.
- **Структура Device/**: код устройств в `Device/<Model>/` (MIC-140 → `Device/MIC140/`, UI → `Device/MIC140/UI/`).
- **Согласование перед написанием кода (RLNX_COLLAB_BEFORE_CODE)**: обязательные вопросы пользователю перед началом реализации новых модулей или логики.
- **Порядок разработки (RLNX_CORE_FIRST_FLOW_2026_05_25)**: разработка функционала строго "от Ядра (Core) к Интерфейсу (UI)".
- **Многопоточность**: правила переключения состояний рабочих потоков, синхронизации через критические секции и использование `suspend`.
- **Lazy Recompile и Рендеринг**: отложенный графический расчет (OpenGL) и обновление UI по таймеру с фиксированной частотой.
- **Плагины (RLNX_CROSSPLATFORM_PLUGINS)**: кроссплатформенная плагинная система на базе Shared Libraries / DLL с Flat C API.
- **Тесты (RLNX_TEST_TYPE_AGREEMENT_2026_05_25)**: правила создания и логирования тест-кейсов в проекте `RecorderTests`.
- **Оригинальный Recorder**: путь к оригинальным исходным кодам Recorder для проверки совместимости.

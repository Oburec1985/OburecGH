# RecorderLnx — кодировка исходников

Связано: [[02_Правила_Разработки]], [[00_Вводная]]

Канонический файл в репозитории: `OburecGH/Lazarus/RecorderLnx/Docs/source-encoding.md`

## Суть проблемы

- Старые `.pas` в git — **CP1251**; `.lfm` — **UTF-8**.
- Неверное сохранение (UTF-8 без перекодировки или AI-diff без кодировки) даёт `???` в комментариях.
- Испорченный файл нельзя «починить» поиском-заменой — нужен git blob в CP1251.

## Решение (2026-06)

1. Перевод UI-модулей на **UTF-8** + `{$codepage UTF8}`.
2. LCL-строки — обычные литералы, без `CP1251ToUTF8`.
3. Восстановление `uMainForm.pas`: скрипт `cach/_fix_umainform_encoding.py` (CP1251 из git → UTF-8).
4. Mojibake в новых диалогах (UTF-8 файл + cp1251): `cach/_fix_oscillogram_settings_encoding.py` — эталон `UI/uRecorderOscillogramSettingsDialog.pas`.

### Симптомы

| В IDE / UI | Причина | Действие |
|---|---|---|
| `???` в комментариях | UTF-8 поверх CP1251 без перекодировки | git CP1251 → `_fix_umainform_encoding.py` |
| `РќР°С…` в Caption | UTF-8 файл + `{$codepage cp1251}` + `CP1251ToUTF8` | `_fix_oscillogram_settings_encoding.py` или ручная миграция |

## Как не ломать снова

- Редактировать кириллицу в **Lazarus IDE** или редакторе с UTF-8.
- Автоправки — только Python с `encoding='utf-8'`.
- Не смешивать `{$codepage cp1251}` и UTF-8 байты в одном файле.
- После правок: нет `???`, `lazbuild -B`, визуальная проверка в IDE.

# MIC-140 в RecorderLnx

Связано: [[00_Вводная]], [[01_Архитектура]], [[08_Карта_Исходников_Оригинального_Recorder]]

## Назначение

MIC-140 подключается в RecorderLnx как источник данных уровня ядра, без DLL-обвязки оригинального Recorder. Цель порта — единый контракт источников данных для виртуальных данных из файла, MIC-140 по Ethernet/TCP и будущих устройств.

Текущий прибор по умолчанию ищется и настраивается как `192.168.14.155:4000`. Поддержаны исполнения на 48 и 96 каналов на уровне конфигурации тегов и источника.

## Выводы по старому драйверу

В старом Recorder есть две полезные ветки:
1. `MIC140_96_rce` — старая модель устройств 48/96 каналов, регистрация классов и Ethernet-интерфейсов, дефолтный адрес `192.168.14.1`.
2. `MebiusDAQDevices\mic140MDQ` — более подходящая для RecorderLnx транспортная модель: устройство работает по Ethernet/TCP через Mebius-пакеты на порт `4000`.

RecorderLnx использует вторую ветку как основу кроссплатформенного транспорта и оставляет 48/96 как параметр конфигурации источника.

## Протокол Mebius/TCP

Реализация в RecorderLnx находится в:

- `D:\works\OburecGH\Lazarus\RecorderLnx\Core\uRecorderMebiusTcpProtocol.pas`
- `D:\works\OburecGH\Lazarus\RecorderLnx\Core\uRecorderMic140DataSource.pas`

Основные константы:

- MEBE signature: `0xA0A0CAFE`.
- Header size: 20 байт.
- IoControl command signature: `0xF10C`.
- Measurement task: `REC_MEASUREMENT_TASK_ID`.
- Data task: `REC_DATA_TRANSMIT_TASK_ID` или старый совместимый `0x3E904000`.
- Порт MIC-140: `4000`.

Команды запуска/останова перенесены как `REC_IOCTL_MEASTASK_START` и `REC_IOCTL_MEASTASK_STOP`. Формат пакета данных MIC-140 разбирается как:

```text
device_id: UInt32
packet_type: UInt16
sample_count: UInt32
data: float[channel, sample]
```

Данные каналов публикуются в `TRecorderTagRegistry.PublishBlock` блоками, а не одиночными точками.

## UI настройки

В `TRecorderSettingsDialog` добавлен маршрут добавления источника:

- кнопка `+` открывает выбор типа источника: `MIC-140` или `Mera file`;
- для MIC-140 открывается диалог с IP, портом, числом каналов 48/96, частотой опроса и таблицей каналов;
- кнопка `?` выполняет автопоиск MIC-140;
- кнопка `...` открывает настройку MIC-140 для выбранного тега/источника;
- выбранные каналы создаются как теги с `ModuleType = MIC-140`, `Address = номер канала`, `PollFrequencyHz = частота`.

Пока канальная настройка сделана прикладным минимумом: включение/отключение каналов, IP/порт, 48/96, частота опроса. Полная матрица режимов оригинального Recorder (`U`, `T`, `R`, тензо, питание датчиков, коммутация, калибровки) ещё не перенесена.

## Автопоиск

В оригинальном Mebius есть UDP broadcast-поиск (`EthernetBus.cpp`). В RecorderLnx добавлена практичная первая версия:

1. TCP-тест соединения с `192.168.14.155:4000`.
2. Параллельный скан `192.168.14.1..254:4000`.
3. Первый найденный адрес подставляется в диалог настройки.

Функции:

- `RecorderMic140TestConnection` — проверка TCP connect.
- `RecorderMic140Discover` — параллельный поиск по подсети.

Следующий шаг — перенести UDP broadcast Mebius detection полностью, если прибор отвечает на `DETECT_BROADCAST` в текущей прошивке.

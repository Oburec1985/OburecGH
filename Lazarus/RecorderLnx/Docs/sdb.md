# SDB: база градуировочных характеристик

## Назначение

SDB (Scales DataBase, БДГХ) -- библиотека готовых градуировочных
характеристик. В оригинальном Recorder это COM-база: пользователь выбирает
запись дерева, драйвер получает объект `IMeScaleEdit`, а затем строит
преобразователь. Для RecorderLnx COM не используется: читается тот же
дисковый формат, поэтому механизм одинаково работает в Windows и Linux.

SDB не является второй системой калибровок. Она импортирует выбранную ГХ в
`TRecorderTagRegistry.Calibrations`; дальнейшее преобразование выполняет
существующая единая цепочка `TRecorderTagRegistry.TransformTagValue`.

## Расположение и формат

Корень определяется `RecorderMeraFilesPath`:

| ОС | Путь по умолчанию |
| --- | --- |
| Windows | `C:\Mera Files\sdb\` |
| Linux | `$MERA_FILES/sdb/`, либо `~/Mera Files/sdb/` |

Структура является деревом каталогов. В корне находится `sdb.xml`; каталог
имеет собственный `*.xml`, а лист -- пара файлов:

* `ГОСТ\Термопары\K_ГОСТ Р 8.585-2001.xml` -- метаданные;
* `ГОСТ\Термопары\K_ГОСТ Р 8.585-2001.csv` -- точки `X,Y`.

XML -- property bag с полями `name`, `dsc`, `module id`, `src from`, `src to`,
`dst from`, `dst to`, `src units`, `dst units`, `state` и `mod time`. Для
термопар `X` -- ТЭДС в мВ, `Y` -- температура в градусах C. Файлы существующей
Mera SDB обычно имеют кодировку Windows-1251; XML reader FPC обрабатывает
кодировку, объявленную в XML-декларации.

`TRecorderSdbTree` строится на `SharedUtils/uBaseObjLaz.pas:TBaseObj`. Папки
SDB -- узлы, а ГХ -- листья `sikScale` с `TSdbScaleInfo`. Эта модель готова для
единого дерева выбора ГХ в диалогах. В свойствах канала MIC-140 кнопка `...`
и двойной щелчок по полю ГХ открывают это дерево. Кнопка `Выбрать` и двойной
щелчок по листу возвращают полный ключ SDB, как `IMeSDBViewer::GetSelected` в
оригинальном Recorder. Обычный комбобокс по-прежнему показывает только ветку
`ГОСТ\Термопары`, как штатный диалог MIC-140.

Пункт `Загрузить из БД ГХ` в общем окне добавления градуировочной
характеристики открывает этот же диалог базы SDB, а не редактор ручной
интерполяции. После выбора характеристика импортируется (или обновляется) в
общем `TRecorderCalibrationList` и назначается редактируемому тегу; отдельной
ветки калибровок для SDB не создаётся.

## Логика MIC-140

1. В свойствах аналогового канала выбирается ГХ из `ГОСТ\Термопары`.
2. В настройки канала сохраняются имя и полный ключ SDB.
3. При подтверждении настройки выбранная CSV-характеристика загружается в
   `TRecorderCalibration` типа `rckPiecewiseLinear` и добавляется или
   обновляется в `Registry.Calibrations` под именем `TC <имя ГХ>`.
4. Тег получает `SourceValueMode=degC`, `UnitName=degC` и эту ГХ в
   `CalibrationNames`.
5. При приёме блока `PublishBlock` вызывает
   `TRecorderTagRegistry.TransformTagValue`; поэтому только выбранный канал
   становится температурным. Каналы без ГХ продолжают отображаться в мВ.

Полный ключ, а не только имя, хранится в описании импортированной ГХ. Это
сохраняет ссылку на источник и позволяет восстановить настройки после
перезапуска проекта.

## Ограничения текущей версии

Поддерживаются табличные SDB ГХ с CSV-точками -- это формат ГОСТ-термопар,
используемый MIC-140. `module id` сохраняется в метаданных и проверяется при
сканировании дерева. Импорт полиномиальных и мультипликативных COM-модулей,
а также запись/экспорт SDB будут добавляться отдельно: они не нужны для
обычного выбора термопары и не должны подменять базовую цепочку калибровок.

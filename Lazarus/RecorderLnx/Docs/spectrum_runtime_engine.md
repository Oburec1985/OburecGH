# Runtime FFT engine for RecorderLnx

Дата: 2026-06-06

## Что реализовано

Добавлен первый core-модуль движка:

```text
Core/uRecorderSpectrumEngine.pas
```

Модуль не зависит от LCL и UI. Его задача - дать чистую runtime-модель, на которую позже можно повесить диалог настройки, сохранение в проектный JSON и вывод результатов на графики.

## Модель настройки

Настройки разделены на два уровня:

- `TRecorderSpectrumConfigNode` - родительский узел конфигурации.
- `TRecorderSpectrumTagBinding` - дочерний тег, который строит спектр по этой конфигурации.

По умолчанию дочерний тег наследует настройки родителя. Если включить `UseOwnSettings`, тег получает собственные параметры. Это повторяет идею `plgControlCyclogram`: меняем родительский узел - настройки применяются ко всем дочерним каналам.

Основные параметры:

- `FFTSize` - размер FFT, степень двойки.
- `Overlap` - перекрытие в отсчетах.
- `SampleRateHz` - частота дискретизации.
- `WindowKind` - Rect/Hann/Hamming/Blackman/FlatTop.
- `NormalizeMode` - None/Max/Energy.
- `KeepPhase` - считать фазу вместе с RMS.

## Runtime-накопитель

Runtime-канал представлен классом:

```pascal
TRecorderSpectrumChannel
```

В него подаются блоки драйвера:

```pascal
FeedSamples(Times, Values, Count)
```

Канал не предполагает, что размер блока драйвера кратен размеру FFT. Он добавляет входные точки в накопитель и вызывает `OnFrame` столько раз, сколько полных окон получилось.

Примеры:

- `FFTSize=8192`, `Overlap=0`, входной блок `16384` отсчетов -> 2 спектра.
- `FFTSize=8192`, `Overlap=4096`, входной блок `16384` отсчетов -> 3 спектра.

Это закрывает случай быстрого качания частоты: если за одну итерацию драйвера накопилось несколько спектральных окон, они будут выданы по очереди, а график сможет положить сразу несколько точек амплитуды/фазы.

## FFT pipeline

`TRecorderSpectrumEvaluator` выполняет один спектр:

1. умножает вход на заранее рассчитанное окно;
2. делает bit-reverse раскладку;
3. выполняет iterative FFT butterfly по заранее рассчитанным twiddle-множителям;
4. считает RMS-амплитуды:

```text
sqrt(Re^2 + Im^2) * sqrt(2) / FFTSize
```

5. опционально считает фазу `ArcTan2(Im, Re)`;
6. применяет нормировку `None`, `Max` или `Energy`.

В горячем цикле нет расчета `SinCos` для FFT-множителей: они строятся один раз в `TRecorderSpectrumFFTPlan`.


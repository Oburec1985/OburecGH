# Заметка по заданию 4

## Что сделано

- Добавлен класс `TSweepSinProgram` в оба варианта `uDacProgram.pas`.
- Режим sweep считает текущую частоту по линейному закону от `StartFrequency` к `EndFrequency` за `SweepTimeSec`.
- В `tests\TestDAC` форма снова поддерживает `Mode = Sin / SweepSin`.
- В `plgControlCyclogram` форма `uDACFrm` теперь поддерживает тот же выбор режима.

## Что проверить руками

1. Открыть `tests\TestDAC`, переключить `Mode` на `SweepSin`, задать `StartFreq/EndFreq/SweepTime`, нажать `Play`.
2. Проверить, что при `Mode = Sin` доступны `Simple Sin / Accurate Sin`, а при `SweepSin` блок выбора программы скрывается.
3. Проверить сохранение и повторную загрузку параметров из INI.
4. При необходимости собрать `plgControlCyclogram` и открыть `uDACFrm`.

## Техническая заметка

- После промежуточной ошибки перекодировки формы .pas были пересобраны заново и сохранены в Windows-1251.
- В обоих .dfm дополнительно исправлены склеенные строки вида nd  object ..., чтобы структура формы снова была текстово корректной.

- По замечанию пользователя форма uDACFrm переделана под штатный паттерн plgControlCyclogram: TRecFrm + фабрика, без отдельного gbSweepSin.

- По последней правке форма DAC обновляет параметры только в активной программе; при переключении режима/типа новая активная программа сразу синхронизируется из UI.
- Уточнена логика формы DAC: UI редактирует только активную программу; при переключении режима/типа новая активная программа сразу получает текущие значения полей.
- Переключение программ на форме DAC переведено на PageControl: вкладки Sin, AccuracySin, SweepSin; активная вкладка определяет единственную активную программу.
- Параметр MinSamples удалён из UI и из AccurateSin: режим точного синуса теперь выравнивает буфер по целому периоду без ручной настройки минимального числа сэмплов.
- Для AccuracySin исправлена генерация буферов: полезная длина блока теперь может быть меньше физического буфера и кратна целому числу периодов; в waveOut отправляется только этот выровненный участок.

# План работ: Анализ памяти, оптимизация отрисовки и правила рефакторинга

## Текущая задача
Исследование повышенного потребления ОЗУ в системе воспроизведения сигналов, исправление падений Access Violation при воспроизведении, устранение лагов отрисовки трендов, добавление правил рефакторинга в системную документацию, а также добавление удаления компонентов по кнопке Del с поддержкой отмены по Ctrl+Z.

## Шаги
- [x] Анализ логики загрузки файлов воспроизведения (`uMeraFile.pas`, `uRecorderDataSources.pas`). Подтверждено потоковое чтение через `TFileStream` вместо полной загрузки файлов в ОЗУ.
- [x] Анализ возможных причин разницы потребления ОЗУ (динамические аллокации в `RefreshTrend` / `EstimateSlice`, фрагментация кучи FPC, кэш ОС).
- [x] Обнаружение и исправление ошибки Use-After-Free с `TMeraSignalInfo` в `uRecorderDataSources.pas`.
- [x] Исправление ошибки Access Violation при вызове `fLastUpdateData.Free;` в `uRecorderTags.pas` (устранено разделяемое поле реестра, публикации событий переведены на безопасные локальные переменные с блоками `try..finally`).
- [x] Оптимизация производительности отрисовки трендов в `uRecorderTrendView.pas` (устранено выделение памяти в куче за счет создания прямого расчета по индексам `CalculateSliceEstimate`, реализован ранний выход `Break` при поиске индексов временного диапазона).
- [x] Добавление удаления компонентов на пользовательских страницах по клавише Del (`VK_DELETE`) в методе `TFormEditorController.HandleKeyDown` модуля `uFormEditorController.pas`.
- [x] Обеспечение возможности отмены удаления по Ctrl+Z (состояние автоматически сохраняется перед удалением через `PushUndoState`, а затем восстанавливается через `RestoreUndoState` при нажатии Ctrl+Z).
- [x] Проверка кодировки `Windows-1251` с CRLF-переводами строк для всех измененных Delphi-модулей.
- [x] Обновление `development-rules.md` (добавлен подраздел "Правила рефакторинга и оптимизации кода" с требованиями по памяти, дублированию кода, константам, парсингу и многопоточности).
- [x] Обновление отчета по оптимизации памяти в файле `notes_memory_optimization.md`.
- [x] Восстановление корректных условий IndexOf (исправление регрессии виртуального девайса) и добавление именования потоков в логе (UIThread, Src_mera.file.1, Src_debug.diagnostics).
- [x] Очистка лог-файла при старте приложения (удаление старого файла лога в секции initialization модуля uRecorderDebugLog.pas).
- [x] Рефакторинг по новым правилам: оптимизация циклов (бинарный поиск в `uRecorderTrendView.pas` и `uRecorderOglOscillogramView.pas` вместо O(N) перебора) и предотвращение перерисовок скрытых элементов (`IsVisible` проверки).
